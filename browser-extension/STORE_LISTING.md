## Long Description

### What Plain Is

Plain is a collection of PDF tools that run entirely in your browser using WebAssembly. Merge, split, extract, reorder, and compress PDFs without uploading files to any server.

This extension provides quick access to Plain.tools from your browser toolbar.

### Why Offline Matters

Traditional PDF tools upload your files to remote servers for processing. This creates privacy risks, especially for sensitive documents like contracts, medical records, or financial statements.

Plain processes files locally. Your documents never leave your device. There is no server, no upload, no third party involved in handling your files.

This can be verified by monitoring network activity in your browser's DevTools while using any Plain tool.

### Who This Is For

- Privacy-conscious users who handle sensitive documents
- Legal, medical, and financial professionals
- Anyone working in restricted network environments
- Users who want PDF tools without accounts or subscriptions
- People who need to process documents offline

### What This Extension Does

This extension is a launcher. It opens Plain.tools pages in new tabs.

**It does not:**
- Process PDFs
- Read files from your computer
- Access browser tabs or history
- Make network requests
- Collect any data

All PDF processing happens on plain.tools in your browser, not in this extension.

## FAQ

### Does this extension upload PDFs?

No. This extension does not handle PDFs at all. It simply opens Plain.tools pages where PDF processing happens locally in your browser. Your files are never uploaded anywhere.

### Does it work offline?

The extension itself works offline (it just opens URLs). The PDF tools on Plain.tools work offline after the initial page load, since all processing uses WebAssembly in your browser.

### Why is it so lightweight?

This extension is intentionally minimal. It requires no permissions, makes no network requests, and stores no data. It exists only to provide quick toolbar access to Plain.tools.

The actual PDF processing happens on the website using pdf-lib and PDF.js, not in the extension.
