## The 30-minute shortlist experiment

Pick one common workflow such as merge or compress and test each candidate with the same sample file set.

### Scoring matrix

| Criterion | Weight | Candidate A | Candidate B | Candidate C |
| --- | --- | --- | --- | --- |
| Upload dependency | 35% |  |  |  |
| Verification clarity | 30% |  |  |  |
| Offline continuity | 20% |  |  |  |
| Operational friction | 15% |  |  |  |

Use a 1-5 score per row and multiply by weight. The highest weighted score is your practical shortlist leader.

## What to observe during testing

- Does processing start only after remote round-trips?
- Is route behaviour obvious to ordinary users?
- Can a reviewer repeat your verification result?

If answers are inconsistent, lower confidence in that route.

## Interpreting results without bias

Avoid common mistakes:

- choosing by brand familiarity
- ignoring route drift risk under deadlines
- giving equal weight to low-impact features

For sensitive workflows, architecture and repeatable verification usually matter more than marginal convenience.

## Recommended decision path

1. Run the matrix with two to three candidates.
2. Keep evidence screenshots for request behaviour.
3. Choose one default route for sensitive files.
4. Re-test quarterly.

This keeps selection practical and auditable.

## Bottom line

The best no-upload PDF tool is the one your team can verify quickly and use consistently without accidental cloud fallback. A structured shortlist test is faster and safer than relying on feature pages alone.
